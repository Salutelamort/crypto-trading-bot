# Идеи из топовых open-source трейдинг-ботов (research-проход)

Цель: взять у популярных ботов с GitHub лучшие **инженерные паттерны** и применить
у себя. Дата обзора: июнь 2026.

## Главная оговорка: звёзды ≠ прибыль

Топовые боты — это **фреймворки/инфраструктура**, а не прибыльные стратегии. Никто
не публикует работающую альфу (была бы — не отдавали бы даром). Поэтому берём
**паттерны** (риск, метрики, валидация, индикаторы), а не «готовые стратегии» из
их README (они почти всегда cherry-picked и переобучены).

Наш движок (эволюция + walk-forward + честный форвард + alpha + диверсификация +
мультитаймфрейм) уже **строже**, чем у большинства этих репо.

## Что у нас УЖЕ есть (не изобретаем заново)

- Комиссии + слиппедж в бэктесте; задержка на закрытый бар (анти look-ahead)
- Walk-forward, честный out-of-sample, метрика consistency
- ATR-стопы как гены, R:R в геноме, кулдаун (= Freqtrade CooldownPeriod)
- Портфельный стоп-кран по просадке (= Freqtrade MaxDrawdown protection)
- Диверсификация: анти-клон по корреляции, квота на символ, лимит промоушена
- Мультитаймфрейм (таймфрейм = ген), макро-фильтр (ETF-потоки), новостной страж
- alpha vs buy&hold, Sharpe

То есть значительная часть «лучшего» у нас уже реализована независимо.

## Что РЕАЛЬНО стоит взять (приоритет)

### 1. Доп. метрики робастности — Sortino, Calmar, Profit Factor  ⭐ВЫСОКИЙ
**Источник:** vectorbt, Freqtrade (стандарт индустрии).
**Идея:** Sharpe штрафует и за «хорошую» волатильность вверх. Дополнить:
- **Sortino** — как Sharpe, но штраф только за просадки вниз (нам важнее).
- **Calmar** — годовая доходность / макс. просадка (прямо про «низкий риск»).
- **Profit Factor** — сумма прибылей / сумма убытков (>1 = прибыльна).
**Куда:** [../src/metrics.py](../src/metrics.py) `compute_metrics`; затем добавить
в отбор/промоушен ([../src/supervisor.py](../src/supervisor.py)) как доп. ворота.
**Почему:** даёт «две независимые метрики», которые сложнее подделать (наш же принцип),
и Calmar буквально про профиль «минимальный риск».

### 2. Freqtrade «protections» — StoplossGuard, LowProfitLock  ⭐СРЕДНИЙ
**Источник:** Freqtrade Protections.
**Идея:** у нас есть кулдаун и MaxDrawdown. Не хватает:
- **StoplossGuard** — если за окно времени случилось N стопов подряд → пауза торговли
  (защита от «рынок сломался, а бот долбит»).
- **LowProfitPairs/Lock** — временно блокировать символ/стратегию с плохим недавним PnL.
**Куда:** [../src/live_trade.py](../src/live_trade.py) (live-страж) +
[../src/risk.py](../src/risk.py). Можно как расширение карантина символов.

### 3. Больше типов стратегий / индикаторов  ⭐СРЕДНИЙ
**Источник:** pandas-ta / ta-lib, наборы стратегий Freqtrade/Jesse.
**Идея:** у нас 12 типов. Кандидаты на добавление (широта поиска эволюции):
- Bollinger Bands mean-reversion / squeeze (есть частично в vol_breakout)
- MACD-кросс, ADX-фильтр тренда (вход только при сильном тренде)
- Keltner channels, Supertrend (ATR-трендследование — близко нашему)
- DCA / Grid (OctoBot) — но это иной класс (усреднение), осторожно с риском
**Куда:** [../src/genome.py](../src/genome.py) (новые типы в STRATEGY_TYPES + signal)
и [../src/indicators.py](../src/indicators.py) (новые индикаторы).
**Осторожно:** больше типов = шире поиск, но и больше шансов на переобучение —
walk-forward + форвард это отсекут, добавлять по чуть-чуть.

### 4. Скорость бэктеста (векторизация)  ⭐НИЗКИЙ (инфра)
**Источник:** vectorbt (NumPy/Numba, «тысячи прогонов за секунды»).
**Идея:** наш бэктест — Python-цикл по барам, медленный на 1h (26k баров).
Векторизовать горячий путь → больше поколений эволюции за тот же облачный бюджет.
**Куда:** [../src/backtest.py](../src/backtest.py). Это рефактор движка — делать
ОТДЕЛЬНО и аккуратно (риск изменить результаты), не во время форвард-эксперимента.

## Что брать НЕ стоит (и почему)

- **Готовые «прибыльные» стратегии из README** — cherry-picked, переобучены, выживут
  единицы; наша эволюция и так их переоткроет честно.
- **Hummingbot маркет-мейкинг** — другой класс (ликвидность/спред), не наш профиль.
- **FreqAI / ML-предсказание цены** — мощно, но «LLM/ML галлюцинируют на сырой
  математике» (принцип треда); добавляет сложность и риск переобучения. Возможно
  позже как ФИЛЬТР входа, не как генератор сделок.
- **Реальная торговля/ключи бирж** — преждевременно, сначала недели форварда.

## Рекомендованный порядок внедрения

1. **Sortino + Calmar + Profit Factor** в метрики и отбор (быстро, безопасно, высокий эффект).
2. **StoplossGuard + блок плохих символов** (усиление риск-слоя).
3. **2–3 новых типа стратегий** (Supertrend, MACD+ADX) — по одному, с проверкой.
4. (позже, отдельно) векторизация бэктеста для скорости.

⚠️ Внедрять **по одному пункту** и смотреть на форвард — не сваливать всё сразу,
чтобы не сбить чистоту текущего эксперимента.
