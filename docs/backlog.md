# Бэклог — отложенные улучшения (НЕ делать посреди форварда без явного «да»)

Каждый пункт здесь отложен сознательно: либо требует накопленного форварда,
либо меняет отбор/риск (запрещено крутить вслепую по CLAUDE.md).

## После накопления форвард-сделок (~несколько недель, 20+ закрытых)

### 1. Пересмотр σ для Deflated Sharpe (SR0)
Сейчас планка SR0 считается по разбросу Sharpe ВСЕХ испытаний (включая заведомо
мусорные случайные геномы). σ раздута → SR0 ≈ 12 при лучшем реальном Sharpe ≈ 5,
sharpe-путь допуска фактически выключен (допуск идёт через alpha/Calmar — система
работает, но один из трёх путей мёртв). Формула Bailey & López de Prado
предполагает испытания одной «семьи» стратегий.
**Идея:** считать σ по верхнему квантилю испытаний или отдельно на семью
(тип×символ), данные уже есть в `agent_stats`.
**Почему не сейчас:** это ОСЛАБИТ планку допуска → больше промоушенов → менять
посреди форварда = слепой тюнинг.

### 1b. Alpha-путь допускает «лучше рынка, но убыточных»
Замечено на e2e 2026-07-02: агент допущен с alpha +11.9% при Sharpe −1.03 и
PF 0.59 — в падающем окне можно обогнать рынок, теряя меньше рынка. Для
профиля «стабильный плюс» стоит рассмотреть доп. условие alpha-пути:
PF > 1 или test_return > 0.
**Почему не сейчас:** ужесточение ворот посреди форварда = тюнинг вслепую;
плюс демоция теперь регулярно переоценивает таких и выкидывает.

### 2. Мета-аллокатор капитала между стратегиями
Детерминированный слой, который взвешивает допущенные стратегии по их ЖИВЫМ
результатам (стабильный плюс на форварде + некоррелированность → больший вес;
поплыла → карантин). Портфель стратегий вместо «кто первый занял слот».
**Почему не сейчас:** взвешивать не по чему — закрытых сделок единицы.

## Инфраструктурная профилактика (по расписанию)

### 3. Чистка истории .git от версий bot.db — повторять ~раз в 1–2 месяца
База коммитится каждые 15 мин; файл лёгкий (retention), но git копит каждую
версию бинарника в истории. Процедура (2026-07-02 выполнена впервые):
бандл-бэкап → `git filter-repo --path data/bot.db --invert-paths` →
вернуть текущий bot.db из облака коммитом → force-push в узкое окно между
облачными прогонами (сорвавшийся прогон сам чинится на следующем цикле).
