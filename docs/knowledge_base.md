# База знаний: как библиотека применена в боте

Документ связывает книги из [../library/](../library/) с конкретными решениями в
коде. Главный принцип (из треда r/algotrading и из самих книг по психологии):

> **Кодируемые принципы → в детерминированный Python. Психология и дисциплина →
> в жёсткие правила системы, которые человек не может нарушить в моменте.**

Бот не «прочитал книги» — в нём реализованы **общеизвестные принципы**, которые
эти книги систематизируют (тексты защищены авторским правом и не копируются).

---

## 1. Стратегии / технический анализ → типы геномов ([../src/genome.py](../src/genome.py))

| Книга | Что применено | Тип генома |
|-------|---------------|-----------|
| Адам Граймс — «Искусство и наука технического анализа» | тренд+откат, failure test, пробой+ретест, сжатие волатильности | `pullback_trend`, `failure_test`, `breakout_retest`, `vol_breakout` |
| Брайан Шеннон — «Мультитаймфреймный технический анализ» | вход только при выстроенном стеке трендов разных горизонтов | `mtf_trend` |
| Р. Вайкофф «Studies in Tape Reading», Д. Вайс «Сделки на горизонте» | пробой, подтверждённый всплеском объёма (объём = топливо движения) | `wyckoff_breakout` |
| Ларри Вильямс — «Секреты торговли на фьючерсном рынке» | пробой волатильности: close ± k·ATR | `williams_volatility` |
| С. Иващенко — «Индикатор RSI» | импульс/перепроданность по RSI | `momentum`, `mean_reversion` |
| Каналы Дончиана (turtle-подход) | трендследование по каналу | `donchian_trend` |

Эти типы — **пространство поиска** для эволюции ([../src/evolution.py](../src/evolution.py)).
Бот автоматически перебирает их (как piratastuertos на фото), а honest out-of-sample
+ метрика consistency отсекают переобучение.

> ⚠️ Дэвид Вайс / Эллиотт (волны) / Price Action курсы дают и более сложные,
> **дискреционные** паттерны (фигуры, разметка волн). Они плохо формализуются в
> детерминированный сигнал без субъективности, поэтому в геном попала только их
> объективная, кодируемая часть (объём+пробой). Это сознательное ограничение,
> а не упущение.

## 2. Управление капиталом и риском → [../src/risk.py](../src/risk.py)

| Книга | Принцип | Где |
|-------|---------|-----|
| Роберт Карвер — «Торговля с кредитным плечом» | размер позиции от волатильности; не рисковать фиксированной долей вслепую | `risk.position_fraction` + стоп по % (основа для ATR-таргетинга) |
| Том Бассо — «Всепогодный трейдер» | система должна выживать в любом режиме; диверсификация по стратегиям | эволюция держит пул разных типов; анти-клон фильтр |
| Кетти Лин — «Секреты проп-трейдинга» | жёсткие лимиты потерь и числа позиций | `stop_loss_pct`, `max_open_positions` |

## 3. Психология трейдинга → встроена как ЖЁСТКИЕ ПРАВИЛА (не как сигналы)

Это самый частый совет всего треда: «бороться с психологией — это жесть»,
«алготрейдинг убирает эмоциональные ошибки». Эти книги **нельзя** превратить в
сигнал, но можно — в правила, которые человек не сможет нарушить в моменте:

| Книга | Урок | Как воплощён в системе |
|-------|------|------------------------|
| Марк Дуглас — «Зональный трейдинг» | думать вероятностями, не отдельной сделкой | отбор по распределению (Sharpe/winrate на десятках сделок), а не по одной |
| Бретт Стинбарджер — «Психология трейдинга 2.0» | процесс важнее результата | супервизор оценивает процесс (consistency), решения логируются с обоснованием |
| Джаред Тендлер — «The Mental Game of Trading» | тильт = нарушение правил после потерь | бот исполняет правила механически, тильт невозможен |
| Стив Уорд, И. Гралеславский (дисциплина) | дисциплина = заранее заданные правила | стоп/тейк/трейлинг зашиты в код, их нельзя «передумать» |
| Карл Ричардс — «Психология инвестиций» | поведенческие ошибки дороги | отсутствие ручного вмешательства в отдельные сделки |

## 4. Контекст / макро / нарратив → дизайн системы и осторожность

| Книга | Что взято |
|-------|-----------|
| Нейт Сильвер — «Сигнал и шум» | отличать сигнал от шума → честный out-of-sample, метрика consistency, фильтр статзначимости (`min_trades`) |
| Елена Чиркова — «Анатомия финансового пузыря» | режимы рынка меняются → kill-switch по просадке, карантин символов |
| Order Flow Trading, «Кому светят японские свечи» | поток ордеров/объём важнее цены в одиночку | объёмный фильтр в `wyckoff_breakout` |
| Дж. Ливермор, Э. Лефевр, Дж. Швагер (Маги рынка) | биографии/интервью — интуиция и контекст | не кодируются; формируют осторожный подход «edge должен найти человек» |

> Книги «Капитал» Маркса и «Атлант расправил плечи» Рэнд — экономическая/
> философская классика, к торговой логике бота отношения не имеют. Оставлены в
> библиотеке как есть; если не нужны — скажи, удалю.

---

## Что это меняет на практике

- Пространство автопоиска выросло с 4 до **12 типов стратегий**, и все они
  обоснованы конкретными книгами, а не взяты с потолка.
- Психология и риск из книг → не «совет на стене», а **исполняемые ограничения**.
- Сам подход остаётся честным: ни одна книга не даёт готовой альфы; бот лишь
  систематически проверяет принципы на out-of-sample данных.
