# Криптовалютный торговый бот

Бот для алгоритмической торговли криптой, построенный по архитектуре из треда
r/algotrading — **особенно по комментарию piratastuertos (тот, что на фото)**.

## Главная идея (с фото)

> «Клод **не выбирает стратегии**. Он решает, стоит ли **убить** агента, который
> работает плохо, стоит ли **продвинуть** одного к реальной торговле и стоит ли
> **генерировать** нового кандидата. Это решения по **управлению**, а не по
> стратегии. Инфраструктура — Python. Уровень управления — Клод. Ни один не
> делает работу другого.»

Поэтому в системе **два чётко разделённых слоя**:

| Слой | Кто | Что делает |
|------|-----|------------|
| **Инфраструктура** | детерминированный Python | данные, индикаторы, бэктест, риск, исполнение — вся математика |
| **Управление** | супервизор (правила или Claude) | kill / promote / generate — только управленческие решения |

Claude (или правила) **никогда** не считает индикаторы и не выбирает отдельные
сделки — потому что LLM галлюцинируют на математике (тоже урок из треда). Торговля
на 100% детерминированная.

## Как это отражает уроки треда

- **«Риск-менеджмент — это всё»** → [src/risk.py](src/risk.py): размер позиции,
  жёсткий стоп, ratcheting trailing stop, take-profit, лимит позиций.
- **«LLM галлюцинируют математику»** → вся математика в чистом numpy/pandas
  ([src/indicators.py](src/indicators.py), [src/backtest.py](src/backtest.py)).
- **«Компилируй вывод LLM в детерминированный код»** → геномы стратегий
  ([src/genome.py](src/genome.py)) компилируются в детерминированные сигналы.
- **«Эволюция: генерировать и убивать тысячи стратегий»** →
  [src/evolution.py](src/evolution.py): генерация, отбор, анти-клон фильтр,
  карантин символов.
- **«Разделяй метрики продвижения и убийства»** + **метрика consistency**
  (train winrate vs test winrate) ловит переобучение →
  [src/supervisor.py](src/supervisor.py), [src/metrics.py](src/metrics.py).
- **«Бумага игнорирует проскальзывание и спред»** → бэктест и бумага учитывают
  комиссию и проскальзывание.
- **«Всегда out-of-sample»** → данные делятся train/test, агенты отбираются
  только по train, честно оцениваются по test.
- **«Внешние данные как guardrail, не как альфа»** →
  [src/macro_feed.py](src/macro_feed.py): потоки в крипто-ETF с farside.co.uk
  как макро-страж risk-on/risk-off.
- **«SQLite для всего»** → [src/db.py](src/db.py).

## Установка

```bash
pip install -r requirements.txt
```

Пакет `anthropic` опционален — нужен, только если хочешь, чтобы управленческие
решения принимал Claude (`supervisor.backend: claude` в конфиге +
переменная окружения `ANTHROPIC_API_KEY`). По умолчанию работают детерминированные
правила — бесплатно и без интернета (как у автора: «$0 на LLM за 60 дней»).

## Запуск

```bash
python main.py fetch      # 1. скачать данные с Binance (публичный API, без ключей)
python main.py evolve     # 2. сгенерировать и отобрать агентов (эволюция)
python main.py supervise  # 3. супервизор: продвинуть/убить агентов
python main.py paper      # 4. форвард-тест продвинутых на out-of-sample
python main.py live       # 5. ЖИВОЙ пейпер в реальном времени (без биржи/ключей)
python main.py status     # состояние популяции
python main.py run        # шаги 1-4 подряд
```

**Живой пейпер** (`live`): торгует в реальном времени, берёт публичные цены с
`data-api.binance.vision` (работает при включённом VPN) и исполняет сделки
локально. Реальных денег и ключей НЕ нужно. Состояние счёта и позиций хранится в
SQLite — бота можно останавливать/запускать без потери позиций. Параметры —
в секции `live` конфига. Если ни один агент не продвинут, при `allow_unpromoted:
true` торгует лучшими кандидатами в ДЕМО-режиме (с пометкой).

## Настройка

Всё в [config.yaml](config.yaml): символы, таймфрейм, параметры риска, издержки,
пороги продвижения/убийства, макро-фильтр.

## ВАЖНО — прочитай перед тем, как думать о реальных деньгах

1. **Это учебный/исследовательский каркас, а не «печатная машинка денег».**
   Главный вывод всего треда: ИИ строит инфраструктуру, но **преимущество
   (edge) должен найти ты сам**. Наивные стратегии (RSI, MA-кроссы) тут есть для
   примера — у них нет преимущества, и система их честно убивает.
2. **Бот торгует только на бумаге.** Реальных API-ключей и реальных ордеров нет.
   Подключение реального брокера — отдельный осознанный шаг после **недель**
   успешного демо.
3. **Бэктест ≠ реальность.** Даже бумага не моделирует дрейф рынка,
   расширение спреда в стрессе и проскальзывание на объёме.
4. **Переобучение — главный враг.** Если крутить параметры до красивой кривой —
   она развалится на реальном рынке. Поэтому здесь мало параметров,
   out-of-sample и метрика consistency.

## База знаний (библиотека книг)

В [library/](library/) — книги по трейдингу, на принципах которых построен бот.
Карта «книга → что применено в коде» — в [docs/knowledge_base.md](docs/knowledge_base.md).

Коротко: пространство автопоиска стратегий выросло до **12 типов**, каждый
обоснован конкретной книгой (Граймс, Шеннон, Вайкофф/Вайс, Ларри Вильямс и др.).
Книги по психологии (Дуглас, Стинбарджер, Тендлер) воплощены **не как сигналы, а
как жёсткие правила**, которые нельзя нарушить в моменте — это и есть главный
смысл алготрейдинга (убрать эмоции). Подробности — в карте знаний.

## Структура

```
main.py              CLI-оркестратор
config.yaml          вся конфигурация
library/             книги (база знаний) + INDEX.md
docs/knowledge_base.md  карта: какая книга во что превратилась в коде
src/
  db.py              SQLite-хранилище (агенты, решения, сделки, свечи)
  data_feed.py       публичные данные Binance (OHLCV, без ключей)
  indicators.py      технические индикаторы (чистая математика)
  genome.py          геном стратегии → детерминированный сигнал
  backtest.py        детерминированный бэктестер (комиссии, слиппедж, стопы)
  risk.py            риск-менеджмент живых позиций
  metrics.py         Sharpe, winrate, просадка, consistency
  evolution.py       генерация/отбор/убийство агентов (+ анти-клон, карантин)
  supervisor.py      УРОВЕНЬ УПРАВЛЕНИЯ (правила или Claude): kill/promote
  macro_feed.py      макро-страж: потоки в крипто-ETF (farside.co.uk)
  paper_trade.py     портфельная бумажная торговля (форвард-тест)
```
