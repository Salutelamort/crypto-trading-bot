# CLAUDE.md — правила работы с этим проектом (ОБЯЗАТЕЛЬНО прочитать перед правками)

Это крипто-трейдинг-бот: **эволюционный отбор стратегий + бумажная (виртуальная)
торговля**. Цель владельца — **минимальный риск + стабильный плюс**, НЕ погоня за
доходностью. Сейчас идёт многонедельный форвард-тест в облаке. Главное правило:
**не сломать то, что уже работает.**

---

## 🔴 КРИТИЧЕСКИЕ ПРАВИЛА (нарушение = поломка проекта)

### 1. Golden-тест после ЛЮБОЙ правки ядра
После изменений в `src/backtest.py`, `src/genome.py`, `src/risk.py`,
`src/metrics.py`, `src/indicators.py` — ОБЯЗАТЕЛЬНО:
```
python tests/golden_backtest.py
```
Должно быть `OK: все 30 геномов совпадают`. Если падает с «РЕГРЕССИЯ» — ты что-то
случайно сломал, чини. Если ты ОСОЗНАННО менял логику расчёта — обнови эталон:
`python tests/golden_backtest.py --update` (и только после того, как убедился, что
новые числа верны). Golden ловит СЛУЧАЙНЫЕ поломки, не намеренные правки.

### 2. НИКОГДА не коммить локальный `data/bot.db`
Облако (GitHub Actions, `.github/workflows/learn.yml`, каждые ~15 мин) ведёт СВОЮ
накопительную forward-историю в `data/bot.db` и коммитит её в `main`. Если
закоммитить локальную базу — затрёшь недели прогресса.
- Коммить **только код** (и `config.yaml`, `docs/`, `tests/`).
- Перед коммитом, если база изменилась локально от тестов:
  `git checkout origin/main -- data/bot.db`
- При конфликте на push — решать **в пользу версии облака** (origin), не локальной.
- Схема БД мигрируется сама при `db.connect` (см. `_migrate` в `src/db.py`), руками
  ничего добавлять не нужно.

### 3. Не трогать фундамент архитектуры
- Торговля 100% **детерминированный Python**. LLM/ИИ **НЕ принимает решения по
  сделкам** — только пишет код. Не вставлять ML/LLM в торговую логику.
- Реальных денег и API-ключей бирж НЕТ. Только публичные данные Binance и
  виртуальный капитал. Не добавлять реальную торговлю без явной просьбы владельца.

### 4. Консервативный профиль риска
Владелец хочет минимальный риск. НЕ повышать экспозицию (`position_fraction`,
`risk_per_trade`, `max_open_positions`, `max_portfolio_drawdown`) без явной просьбы
И без доказательства на форварде. Сначала форвард доказывает плюс — потом масштаб.

### 5. Мусорные файлы
Иногда в корне появляются пустые (0 байт) файлы со странными именами (`int`, `up`,
`{thresh}`, `0)` и т.п.) — это случайные консольные редиректы. Удалять (только
0-байтные!), в коммит не включать.

## Как это работает (карта кода)

| Файл | Роль |
|------|------|
| `daily_learn.py` | Точка входа ОБЛАКА: fetch → evolve (пачкой) → supervise → live tick → журнал. В конце чистит свечи (база лёгкая). |
| `main.py` | CLI: `fetch / evolve / supervise / paper / live / status / run`. |
| `src/genome.py` | 14 типов стратегий + гены риска (stop_atr, rr, trail_atr, cooldown). signal() → {-1,0,+1}: short/кэш/long. |
| `src/backtest.py` | Детерминированный бэктест (long/short/кэш, ATR-стопы, кулдаун, vol-таргетинг). walk_forward_eval с embargo. |
| `src/evolution.py` | Эволюция: генерация/мутация/отбор по квоте на символ, анти-клон по корреляции дохода. |
| `src/supervisor.py` | Допуск к торговле: Sharpe(с Deflated-планкой) ИЛИ alpha ИЛИ Calmar + диверсификация. |
| `src/risk.py` | Position (long/short), уровни выхода (ATR), vol-таргетинг сайзинга (`sized_fraction`). |
| `src/protections.py` | StoplossGuard (пауза после серии убытков) + блок плохих символов. |
| `src/metrics.py` | Sharpe/Sortino/Calmar/ProfitFactor/alpha + `expected_max_sharpe` (Deflated Sharpe). |
| `src/live_trade.py` / `src/paper_trade.py` | Бумажная торговля (live tick / портфельный прогон). Реальных денег НЕТ. |
| `src/data_feed.py` | Публичные данные Binance (без ключей), мультитаймфрейм. |
| `config.yaml` | Все параметры. Мультитаймфрейм: 1h/2h/4h/6h/8h/12h/1d. |
| `tests/golden_backtest.py` | Регрессионный эталон ядра (30 геномов). Запускать после правок ядра. |
| `docs/competitors_ideas.md` | Что взято/не взято из топ-ботов (раунды 1-2 закрыты). |

## Что уже сделано (НЕ переделывать без причины)
Три состояния long/short/кэш · alpha-метрика · ATR-стопы как гены · кулдаун ·
мультитаймфрейм · диверсификация (анти-клон по корреляции + квота на символ + лимит
1 позиция/символ) · метрики Sortino/Calmar/PF · protections · стратегии
supertrend/macd_adx · оптимизация бэктеста · volatility-targeting сайзинга ·
Deflated Sharpe · embargo. Все идеи из обзора репозиториев (раунды 1-2) ВНЕДРЕНЫ.

## Текущий статус
**Форвард-тест идёт в облаке** на консервативных настройках. Задача — ждать недели
и копить живую историю (`TRACK_RECORD.csv`), НЕ перетюнивать вслепую. Следующий
осмысленный шаг — анализ форвард-сделок из `data/bot.db` (после накопления), потом
точечный тюнинг по фактам. Не «крутить риск и надеяться».
